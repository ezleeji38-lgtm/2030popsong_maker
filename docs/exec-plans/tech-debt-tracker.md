# Tech Debt Tracker

## 개요

개발 과정에서 발생하는 기술 부채를 추적한다. 각 항목은 발생 시점, 원인, 해결 계획을 기록한다.

---

## 예상 기술 부채

### TD-001: Suno 비공식 API 의존

```
심각도: 높음
발생 시점: Phase 2 (곡 생성 구현 시)
원인: Suno 공식 API 부재 → 서드파티 래퍼(suno-api) 사용
리스크: 래퍼 중단, Suno 정책 변경, 인증 방식 변경
해결 계획: 어댑터 패턴으로 격리. 공식 API 출시 시 어댑터만 교체.
상태: 미발생 (개발 전)
```

### TD-002: Gemini 프리뷰 모델 사용

```
심각도: 중간
발생 시점: Phase 2 (이미지 생성 구현 시)
원인: gemini-3.1-flash-image-preview는 프리뷰 모델
리스크: 모델 ID 변경, API 인터페이스 변경, 가격 변동
해결 계획: 모델 ID를 config.toml에서 관리. fallback 모델 설정.
상태: 미발생 (개발 전)
```

### TD-003: FFmpeg subprocess 직접 호출

```
심각도: 낮음
발생 시점: Phase 3 (렌더링 구현 시)
원인: Python FFmpeg 래퍼 대신 subprocess 직접 호출
리스크: 명령어 조합 복잡도 증가, 에러 메시지 파싱 어려움
해결 계획: 당분간 유지. 렌더링 옵션 증가 시 ffmpeg-python 래퍼 도입 검토.
상태: 미발생 (개발 전)
```

### TD-004: JSON 파일 기반 데이터 저장

```
심각도: 낮음
발생 시점: Phase 1 (기반 구축 시)
원인: 개인 도구로 SQLite 대신 JSON 파일 선택
리스크: 곡이 수백 개 이상으로 증가 시 목록 조회 성능 저하
해결 계획: 당분간 유지. 성능 이슈 발생 시 SQLite 전환.
상태: 미발생 (개발 전)
```

### TD-005: Suno API 불안정 + 캡차 + 쿠키 갱신

```
심각도: 높음
발생 시점: Phase 2 (곡 생성 구현 시)
원인:
  - suno-api 래퍼의 500 에러 빈발
  - hCaptcha: 유료 계정도 면제 안 됨 → 2Captcha(유료) 필수
  - 쿠키: ~7일마다 만료 → 수동 갱신 필요
  - Windows에서 캡차 빈도 높음
리스크:
  - 2Captcha 잔액 소진 시 곡 생성 불가
  - 쿠키 만료 미인지 시 파이프라인 실패
  - 전체 파이프라인 신뢰성 저하
해결 계획:
  - 대안 경로 3종 (수동 MP3 임포트, 서드파티 API 전환, 다른 AI 음악 서비스)
  - 실행 전 2Captcha 잔액/쿠키 유효성 사전 체크
  - 쿠키 만료 임박 시 CLI 경고 메시지
상태: 미발생 (개발 전)
```

## 기록 형식

```
### TD-{번호}: {제목}

심각도: 높음 / 중간 / 낮음
발생 시점: Phase X (구체적 작업)
원인: 왜 이 부채가 생겼는가
리스크: 방치 시 어떤 문제가 발생하는가
해결 계획: 언제, 어떻게 해결할 것인가
상태: 미발생 / 발생 / 해결됨
```
