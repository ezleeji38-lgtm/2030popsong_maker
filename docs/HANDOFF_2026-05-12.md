## 3. 남은 작업 — 다음 세션에서 이어가기

### A. 운영 4단계 (사용자만 가능 — 브라우저 작업)

**상태: 미완료, 다음 세션 시작점.**

| 단계 | 작업 | 링크/방법 | 결과물 |
|---|---|---|---|
| 1.1 | Google Sheets API 활성화 | https://console.cloud.google.com/apis/library/sheets.googleapis.com | "사용 설정됨" 표시 |
| 1.2 | Google Drive API 활성화 | https://console.cloud.google.com/apis/library/drive.googleapis.com | "사용 설정됨" 표시 |
| 1.3 | Service Account 생성 | https://console.cloud.google.com/iam-admin/serviceaccounts | `songmaker-sa@suno-song-xxxxx.iam.gserviceaccount.com` 이메일 발급 |
| 1.4 | SA 키 JSON 다운로드 → 이동 | 위 SA 클릭 → 키 탭 → 새 키 만들기 → JSON | `~/.songmaker/service_account.json` 에 배치 |
| 2 | 페르소나 시트 생성 + SA 공유 | https://sheets.google.com → 새 시트 → 공유 → SA 이메일 편집자 권한 | 시트 ID (`/d/<여기>/edit`) |
| 3 | songmaker config로 시트 ID 등록 | `songmaker config` | `~/.songmaker/config.toml` `[sheets] default_sheet_id` 채워짐 |

**진행 끝나면 검증**:
```
songmaker doctor
```
→ 모든 항목 ✓이어야 함.

### B. 운영 5-6단계 (4단계 끝나면 자동 진행 가능)

```bash

# 5-2. 컨셉 15개 시트로 일괄 푸시

# 6-1. 가사 변환 (5규칙)
songmaker transform-batch

# 6-2. 사용자가 B열을 "DO IT"으로 변경 (15행 트리거)
```
