# Song Maker — Product Sense

## 1. 제품 비전

"트렌드를 읽고, 곡을 만들고, 영상으로 올리는" 전 과정을 하나의 CLI 명령으로 자동화한다.

---

## 2. 핵심 가치

| 가치 | 설명 |
|------|------|
| **자동화** | 반복 작업(조사 → 생성 → 렌더링 → 업로드)을 한 번에 처리 |
| **트렌드 기반** | 감이 아닌 데이터(YouTube 인기 차트)에 기반한 곡 방향 결정 |
| **사용자 통제** | 자동화 속에서도 핵심 의사결정(주제, 장르, 분위기)은 사용자가 한다 |
| **단순한 인터페이스** | CLI 명령어 하나로 전체 파이프라인 실행 가능 |

---

## 3. 사용자 시나리오

### 시나리오 A — 전체 파이프라인

```
1. 사용자: songmaker run
2. 시스템: 유튜브 트렌드 조회 → 테이블 표시
3. 사용자: 장르=발라드, 분위기=몽환적, 주제="봄날의 설렘" 입력
4. 시스템: Suno로 곡 생성 (대기) → 완료
5. 시스템: Gemini로 배경 이미지 + 썸네일 생성
6. 시스템: FFmpeg로 영상 렌더링
7. 시스템: YouTube에 비공개 업로드
8. 사용자: 결과 확인 → 공개 전환
```

### 시나리오 B — 단계별 실행

```
1. songmaker trend         → 트렌드만 확인
2. songmaker create        → 곡만 생성
3. songmaker image abc123  → 이미지만 생성
4. songmaker render abc123 → 렌더링만
5. songmaker upload abc123 → 업로드만
```

### 시나리오 C — 배치 생성

```
1. songmaker run --count 3
   → 같은 방향으로 3곡 생성
   → 3개 영상 렌더링
   → 3개 영상 업로드
```

---

## 4. 의사결정 원칙

| 상황 | 결정 |
|------|------|
| 자동화 vs 사용자 선택 | 창작 방향은 사용자 선택, 기술적 처리는 자동화 |
| 품질 vs 속도 | 기본은 속도 우선 (flash 모델), 고품질 옵션 제공 |
| 비공개 vs 공개 | 항상 비공개로 먼저 업로드, 확인 후 공개 |
| 단일 vs 배치 | 단일이 기본, 배치는 옵션 |

---

## 5. 하지 않는 것

| 항목 | 이유 |
|------|------|
| 웹 UI | CLI로 충분, 복잡성 증가 방지 |
| 사용자 계정 시스템 | 개인 도구, 불필요 |
| 곡 편집 기능 | Suno에서 재생성이 더 효율적 |
| 실시간 스트리밍 | 범위 밖 |
